# Documentation

This directory drives the POC creation pipeline.

## How to use

1. Drop your PRD (Product Requirement Document) files as `.md` into `docs/prds/`
2. Open the project in Claude Code
3. Say one of:
   - **"run full pipeline"** — executes all phases end-to-end
   - **"analyze PRDs"** — Phase 1 only: refine PRDs into development artifacts
   - **"research UI"** — Phase 2 only: find design inspiration
   - **"build it"** — Phase 3+4: develop and validate in a loop
   - **"deploy"** — Phase 5: deploy to GitHub Pages (default)
   - **"deploy to Vercel"** — Phase 5: deploy to Vercel (requires account)

## Directory structure

```
docs/
├── prds/                  ← Your PRD files go here (.md format)
├── artifacts/             ← Generated by Phase 1
│   ├── user-stories/      ← User stories with acceptance criteria
│   ├── component-specs/   ← React component tree and interfaces
│   ├── api-specs/         ← API endpoints and data shapes
│   └── data-models/       ← TypeScript interfaces and data flow
└── design-references/     ← Generated by Phase 2
    └── design-system.md   ← Unified design tokens and patterns
```

## PRD format

PRDs can be in any markdown format. The more detail you provide, the better the output. Include:
- Problem statement
- Target users
- Key features / requirements
- Success metrics (optional)
- Constraints (optional)
